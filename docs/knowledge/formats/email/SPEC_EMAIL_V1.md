# SPEC\_EMAIL\_V1.md

SPEC\_ID: EMAIL\_NEWSLETTER\_V1
USE\_WHEN: email рассылка, email-рассылка, письмо, дайджест, рассылка SimpleWine
FORMAT\_TAGS: email, рассылка

## Goal

Сформировать комплект текстов для email-рассылки в заданном шаблоне и с вариантностью.

## Hard constraints

### 1\) Лимиты длины

* Тема: ≤ 30 символов.
* Прехедер: ≤ 75 символов.

### 2) Объём текста блоков
* Текст каждого смыслового блока (описание/лид/тело блока): максимум 1–2 абзаца.
* Никаких “мини-статей” и длинных цитат: оставлять только ключевые факты/выгоды из брифа.


### 2\) Недопустимое дублирование частей письма

Части письма:

* Тема
* Прехедер
* Заголовок баннера
* Подзаголовок баннера
* Вводный текст
* Заголовки блоков

Правила:

* Запрещены дословные совпадения фрагментов длиной 3+ слов подряд между любыми двумя частями.
* Запрещено точное совпадение строк:

  * Тема = Заголовок баннера
  * Прехедер = Подзаголовок баннера

* Одна мысль допустима, но под разным углом:

  * Тема: рамка/интрига/настроение
  * Прехедер: конкретика выгоды/суть предложения
  * Заголовок баннера: образ/сцена/обещание
  * Подзаголовок: уточнение «что внутри»
  * Вводный: знакомый сценарий/почему сейчас/как это работает/краткое саммари, что будет дальше

### 3\) CTA и соответствие ссылке

* Текст CTA обязан соответствовать странице, куда ведёт ссылка.
* Если ссылка ведёт на главную, CTA не должен обещать формат, которого там нет (например, «дайджест/обзор/статья/читать» — если это не так).
* Запрещено использовать «купить» (кроме «купить в винотеке»).

Разрешённый базовый набор CTA (можно расширять при необходимости):

* «Заказать»
* «Выбрать»
* «К подборке»
* «К винам» / «К игристому» / «За крепким»
* «Собрать корзину»
* «Найти свое»
* «Подробнее»

### 4\) Output contract обязателен

Если где-либо не хватает требуемого числа вариантов — ответ недействителен: перегенерируй недостающие части и перепроверь.

## Preferences

* Тема — цепляющий заголовок-рамка (сцена/сезон/интрига/эмоция).
* Прехедер — максимум конкретики; самое цепляющее ближе к началу (выгода/подарок/условия/редкость/конкретика по винам/регионам).
* «Скидки» — использовать дозированно; предпочитать вариативные формулировки выгоды.

## OUTPUT\_CONTRACT (строгий шаблон вывода)

Всегда выдавай в таком порядке:

1. ТЕМА — 3 варианта
2. ПРЕХЕДЕР — 3 варианта
3. ГЛАВНЫЙ БАННЕР:

   * Заголовок — 3 варианта
   * Подзаголовок — 3 варианта

4. ВВОДНЫЙ ТЕКСТ — 1 абзац (1 вариант) + кнопка CTA(3 варианта)
5. (Если есть) ЗАГОЛОВОК ПОДБОРКИ ТОВАРОВ — 3 варианта
6. ДЛЯ КАЖДОГО БЛОКА:

   * Заголовок блока — 3 варианта
   * Текст блока — 1 вариант
   * Кнопка/CTA блока — 3 варианта

Правило вариантности:

* Варианты должны быть независимы: меняй не 1 слово, а угол подачи (сцена / выгода / интрига / конкретика).

## PREFLIGHT\_CHECKS (внутренне, не показывать)

* Длины соблюдены (≤ 30 / ≤ 75).
* Output contract выполнен: везде, где требуется, есть 3 варианта.
* Нет дословных совпадений 3+ слов между частями; нет точного равенства строк (Тема≠Заголовок; Прехедер≠Подзаголовок).
* CTA соответствует ссылке и не противоречит странице (особенно если ссылка на главную).
* Заголовок блока и CTA соответствуют содержанию блока (не сужают смысл до одной категории, если в блоке несколько).
* Нет подряд 3 слов с одинаковым корнем (особенно «вин… вин… вин…»).
* Нет запрещённых слов/клише из SW\_LEXICON\_AND\_BANS\_V1.
* Нет “ярлыков регионов” из SW\_GLOBAL\_RULES\_V1.
* Нет служебных фраз в ответе.
* Каждый блок: текст ≤ 2 абзацев (если длиннее — сжать, сохранив факты).
